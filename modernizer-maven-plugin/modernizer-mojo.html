modernizer:modernizer

Full name:

org.gaul:modernizer-maven-plugin:2.4.0-SNAPSHOT:modernizer

Description:

(no description)

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: process-test-classes.

Optional Parameters

Name Type Since Description
exclusionPatterns Set - Violation patterns to disable. Each exclusion should be a regular expression that matches the javap format: java/lang/.*
exclusions Set - Violations to disable. Each exclusion should be in the javap format: java/lang/String.getBytes:(Ljava/lang/String;)[B.
exclusionsFile String - Disables user-specified violations. This is a text file with one exclusion per line in the javap format: java/lang/String.getBytes:(Ljava/lang/String;)[B.
User property is: modernizer.exclusionsFile.
failOnViolations boolean - Fail phase if Modernizer detects any violations.
Default value is: true.
User property is: modernizer.failOnViolations.
ignoreClassNamePatterns Set - Fully qualified class names (incl. package) to ignore by regular expression, specified using <ignoreClassNamePattern> child elements. Specifying .*.bar.* ignores foo.bar.*, foo.bar.baz.* but also bar.* and so on; or .*Immutable ignores all class with names ending in Immutable in all packages.
ignoreGeneratedClasses Boolean - Classes annotated with @Generated will be excluded from scanning.
Default value is: true.
User property is: modernizer.ignoreGeneratedClasses.
ignorePackages Set - Package prefixes to ignore, specified using <ignorePackage> child elements. Specifying foo.bar subsequently ignores foo.bar.*, foo.bar.baz.* and so on.
includeTestClasses boolean - Run Modernizer on test classes.
Default value is: true.
User property is: modernizer.includeTestClasses.
javaVersion String - Enables violations based on target Java version, e.g., 1.8. For example, Modernizer will detect uses of Vector as violations when targeting Java 1.2 but not when targeting Java 1.1.
User property is: modernizer.javaVersion.
outputDirectory File - The output directory into which to find the resources.
User property is: project.build.outputDirectory.
skip boolean 1.4.0 Skips the plugin execution.
Default value is: false.
User property is: modernizer.skip.
sourceDirectory File - The output directory into which to find the source code.
User property is: project.build.sourceDirectory.
testOutputDirectory File - The output directory into which to find the test resources.
User property is: project.build.testOutputDirectory.
testSourceDirectory File - The output directory into which to find the test source code.
User property is: project.build.testSourceDirectory.
violationLogLevel String - Log level to emit violations at, e.g., error, warn, info, debug.
Default value is: error.
User property is: modernizer.violationLogLevel.
violationsFile String - User-specified violation file. Also disables standard violation checks. Can point to files from classpath using an absolute path, e.g.: classpath:/modernizer.xml for the default violations file.
User property is: modernizer.violationsFile.
violationsFiles List - User-specified violation files. The violations loaded from violationsFiles override the ones specified in violationsFile (or the default violations file if no violationsFile is given). Violations from the latter files override violations from the former files. Can point to files from classpath using an absolute path, e.g.: classpath:/modernizer.xml for the default violations file.
User property is: modernizer.violationsFiles.
